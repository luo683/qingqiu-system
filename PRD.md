## 1. 一句话定位

**清秋 = 一个由本地语音驱动、被动响应、权限严格、接 Obsidian 知识库的私人 AI 助理**。它通过 Hermes 任务总线持续学习你、读你的笔记、看你的知识图谱，对你越来越懂；对外只露一个名字"清秋"，对内由可热插拔的 LLM 大脑驱动。

不是聊天机器人，不是 IDE 插件，不是云端 SaaS —— 是**一个装在你机器上、看过你所有笔记、记得你所有偏好、但要你点头才动手的私人中枢**。

## 2. 设计哲学（八条原则）

| # | 原则 | 含义 |
|---|------|------|
| P1 | **用户主权** | 一切以"owner = 你"为中心。清秋不主动猜、不主动说、不主动做。 |
| P2 | **可换大脑** | LLM 是可热插拔的抽象层。今天 OpenAI，明天 Claude，后天本地 Ollama。 |
| P3 | **本地优先** | 语音、记忆、知识库、任务数据全部本地。LLM 可走云，但默认支持纯本地。 |
| P4 | **被动响应** | 不在你喊时说话。反 JARVIS 常识但尊重你的决定。 |
| P5 | **写入前必问** | 任何写操作（文件、邮件、IM、git push、vault 笔记）必须先拿到你明确确认。 |
| P6 | **可解释** | 每个决策可追溯到 Planner 拆解 + 你的原话。出错能复盘。 |
| P7 | **人格可调** | 不绑定固定人设。清秋说什么、怎么说，由你用 prompt 自己定义。 |
| P8 | **私密信息零泄露** | 身份、财务、健康、凭证、私密文件、笔记中的私密段落 —— 默认拒绝处理；处理需你逐项明示同意。 |

## 3. 从 Hermes 到清秋 —— 迁移故事

Hermes 是**开发态底座**：手工写 task JSON、手工跑 handoff-engine.py。清秋要把它**吸收重塑**：

| Hermes（v0.1） | 清秋（v1.0） | 变化 |
|----------------|--------------|------|
| 三个固定 agent（Codex / OpenCode / ZCode） | **按场景可注册的角色池** | 不再写死 |
| 文件总线 + lock 抢占 | **任务总线（保留）+ Planner 增强** | 协议向上兼容 |
| 手工 `python handoff-engine.py create` | **Planner 自动调度** | 触发自动化 |
| 路径写在 hermes-config.json | **清秋统一管理 workdir** | 路径对齐 |
| 没有 owner / 没有权限控制 | **owner=user + 写入确认 + 目录白名单 + 私密识别** | 加安全 |
| 没有语音入口 | **语音优先入口（faster-whisper + piper）** | 加通道 |
| 没有记忆系统 | **四层记忆体系** | 加持续性 |
| 没有 LLM 抽象 | **可热插拔 LLM provider** | 加灵活性 |
| 没有 prompt 人格 | **用户自定义 system prompt** | 加个性化 |
| **没有知识库接入** | **接 Obsidian vault + 知识图谱可视化** | 加知识 |
| **任务历史只用于审计** | **任务历史反哺自我成长** | 加学习 |

**核心承诺：** 现有 Hermes 仓库的所有 task schema、所有 in-flight 任务、所有 agent 实现代码 → 都能在清秋中继续跑，只是入口从"手写"变成"喊一声"。

## 5. 范围（v1.0 做 / 不做）

### 5.1 做

- ✅ 本地语音入口（faster-whisper STT + piper TTS，完全离线）
- ✅ 被动响应模型
- ✅ 多模态输入（文本 / 语音 / 图片 / PDF / 视频 / 文件元数据）
- ✅ 五类 Agent 角色（coder / reviewer / info / life / knowledge）
- ✅ 可热插拔 LLM provider
- ✅ 标准白名单目录（4 个目录读写）
- ✅ 每次写入前确认
- ✅ 四层记忆体系
- ✅ prompt 自定义人格
- ✅ 多入口并行（语音主入口 + 文字 CLI + 飞书 IM 补充入口）
- ✅ Obsidian vault 接入（路径在 config 配置）
- ✅ 知识图谱可视化（vault 节点 / 双向链接 / 标签 / 聚类）
- ✅ 自我成长机制（Hermes 任务历史反哺 + 偏好学习 + vault 反哺）
- ✅ 每周复盘自动生成（写到 `~/.qingqiu/memory/weekly/`）

### 5.2 不做

- ❌ 多用户 / 多租户
- ❌ 主动通知 / 定时汇报 / 异常主动说话
- ❌ 云端部署 / 公网暴露 / Web UI
- ❌ 移动端独立 App
- ❌ 自动消费级硬件控制
- ❌ 模型自训练 / 微调
- ❌ Obsidian 之外的笔记软件接入
- ❌ 知识图谱的语义理解（v1 只做结构性图谱）
- ❌ 生产级密钥管理 / 审计日志 / 合规

## 6. 数据与权限（含私密信息保护 · P8）

### 6.1 标准白名单

```
✅ 可读：
   E:\MiniMax Code WorkSpace\
   C:\Users\ROG\Downloads\
   C:\Users\ROG\Desktop\
   C:\Users\ROG\Documents\

❌ 不可读：
   C:\Windows\
   C:\Program Files\
   C:\Users\ROG\AppData\
   其他目录（除非显式加入白名单）
```

### 6.2 写入策略（P5 · 写入前必问）

每个写操作前必须经 `Confirm` 模块：

- **文字：** CLI/TUI 弹 `? Apply these 3 changes? [y/N/diff]`
- **语音：** TTS 念出"我要改 3 个文件，是否继续？"
- **IM：** 飞书消息带按钮 `[应用] [拒绝] [看 diff]`
- **超时：** 60 秒不响应 → 自动拒绝

### 6.3 危险操作黑名单

- `git push --force`
- `rm -rf`
- `format`
- 改系统配置
- 发送邮件 / IM（含私密信息走 §6.4 例外通道，逐条确认）
- **上传任何文件到云端 / 第三方服务**
- **跨白名单目录移动文件**
- **读取私密文件 / 含凭证的文件**
- **导出记忆库 / 用户偏好文件**
- 🆕 **批量修改 vault 笔记 / 删除 vault 文件**

### 6.4 私密信息保护（P8 · 零泄露）

**三道防线：**

1. **识别（Detect）：** 文件名模式 + 内容正则 + 目录黑名单
2. **阻断（Block）：** 命中 → 中止任务 → 告知用户 → 处置方式
3. **脱敏（Redact）：** 占位符替换，脱敏映射本地存

**三种例外通道：** `include_private` / `redact_only` / `private_send`（全部限时 + 逐条）

### 6.5 Obsidian Vault 接入

- **路径：** `~/.qingqiu/config.yaml` 配置 `obsidian.vault_path`
- **索引：** 启动全量 + watchdog 增量
- **私密笔记：** frontmatter `private: true` 或 `private/` 子目录默认跳过
- **写入：** 走 Confirm 模块；删除二次警告（不可逆）

## 7. 主动性与会话模型

### 7.1 主动性：纯被动（P4）

- 不在你喊时说话
- 不做定时汇报
- 不做异常主动通知
- **任务完成通知也默认关闭**

### 7.2 会话模型

| 模式 | 触发 | 终止 |
|------|------|------|
| 单轮 | 一句话一个任务 | 任务结束 → 关闭 |
| 多轮 | 复杂任务需要追问 / 确认 | 你说"结束"或超时 5 分钟 |
| 持续会话 | 显式开启 | 你说"结束" |

## 8. 记忆与人格

### 8.1 四层记忆

| 层 | 存储位置 | 用途 |
|----|---------|------|
| **L0 会话内** | 内存 | 当前对话上下文 |
| **L1 项目级** | `~/.qingqiu/memory/projects/<name>.md` | 项目背景 / 约定 |
| **L2 用户级** | `~/.qingqiu/memory/user.md` | 偏好 / 习惯 / 核心概念 |
| **L3 长期事实** | `~/.qingqiu/memory/facts.sqlite` | 历史任务 / 习惯统计 / 时间线 |

### 8.2 人格（prompt 自定义 · P7）

```yaml
# ~/.qingqiu/personality.yaml
personality:
  name: 清秋
  system_prompt: |
    你是清秋，给 ROG 个人使用。
    风格：简洁、直接、技术男、偶尔幽默。
    唤醒词："清秋"。
```

## 9. 自我成长机制（5 条学习路径）

1. **任务归档 → 长期事实**
2. **用户纠正 → 偏好固化**
3. **vault 反哺 → 概念库**
4. **每周复盘 → 长期记忆**（周日 23:00）
5. **偏好冲突检测 → 询问而非自作主张**

**学到的边界：**

- ❌ 不主动学（不偷偷读你的文件"自我提升"）
- ❌ 不替你做决策
- ❌ 不联网学
- ❌ 不写 personality.yaml 之外的配置文件

## 10. 接入与多模态

### 10.1 入口优先级（v1.0）

## 11. 安全

| 风险 | 防护 |
|------|------|
| **写越界** | Confirm + 白名单 + 黑名单 |
| **Token 泄露** | 系统 keyring，不入仓 |
| **Prompt 注入** | 用户输入与外部内容严格隔离 |
| **任务卡死** | 任务超时 + 自动 fail |
| **离线兜底** | 网络断开切本地 LLM |
| **录音泄露** | 24h 自动清除 |
| **私密信息泄露** | 三道闸 + LLM 路由屏蔽 + 记忆白名单 |
| **vault 私密笔记泄露** | frontmatter + private 目录双轨跳过 |
| **LLM 用 vault 内容训练** | 全部 provider opt-out |

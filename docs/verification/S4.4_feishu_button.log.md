## 1. 设计要点

飞书消息类型：
- `text`：纯文本（M4 已实装）
- `interactive`：卡片消息（v2 schema）— **S4.4 实装**

卡片 3 类：
1. `from_text`：纯文本卡片（无害信息卡）
2. `confirm_card`：**Confirm / Cancel 双按钮**（v1.0 MVP 主脑风险流程用）
3. `info_card`：只读信息卡（无按钮）

## 6. 集成点

### v1.0 confirm-then-execute 流程

```
用户 (飞书): memory set secret xxx
↓
Executor.execute() (S5.5 sensitive Block)
↓ 触发 confirm
IM Handler (S4.2)
↓ 调 client.send_interactive()
↓
飞书 → 用户 confirm 卡
↓
用户点 ✓ 确认
↓
按钮 click → dispatch()
↓
Executor.execute() 继续（已 confirm）
↓ 回发结果
用户收到回复
```

### 接 `confirm_card()` 自动检测

主脑对 risky 操作自动生成 confirm card：
- memory set sensitive (S5.5)
- file write (S5.2 whitelist)
- 任意 destructive intent

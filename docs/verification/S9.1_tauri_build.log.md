## 2. 构建过程

```bash
# Step 1: 验证 Rust 工具链（已装）
rustc 1.96.0

# Step 2: 装 tauri-cli ~5 分钟
cargo install tauri-cli --version "^2.0" --locked
# → cargo-tauri v2.11.4

# Step 3: icon 自动生成（从 1024x1024 PNG）
uv pip install pillow
python -c "from PIL import Image, ImageDraw; ..."
cargo tauri icon src-tauri/icons/icon.png
# → 自动生成 icon.ico, icon.icns, 32x32.png, 64x64.png, 128x128.png,
#   128x128@2x.png, StoreLogo.png, Square*Logo.png (Win/UWP),
#   AppIcon-*.png (iOS), ic_launcher*.png (Android)

# Step 4: 前端 dist stub（因为 M9 用 FastAPI 而非静态打包）
mkdir web/dist
Copy-Item web/index.html web/dist/index.html

# Step 5: 修 2 个 tauri 编译错
- Cargo.toml [lib] block → 删（不要 lib mode）
- build.rs 加 fn main() { tauri_build::build(); }
- tauri.conf.json trayIcon.iconPath → 改 icons/32x32.png（iconPath 用生成路径，无 tray.png）
- icons/32x32.png 自动生成，不用画

# Step 6: 编译 + bundle
cargo tauri build
# ✓ Compiling 成功 (1m 31s release)
# ✓ MSI 生成（wix 3.x 自动下载）
# ✓ NSIS 生成（makensis 自动下载）
```

## 6. 关键决策（D-045）

| # | 决策 | 原因 |
|---|------|------|
| 1 | 删 Cargo.toml `[lib]` block | 单 bin，不需要 lib mode |
| 2 | 加 `build.rs` | tauri::generate_context! 需要 OUT_DIR（build script 产出） |
| 3 | trayIcon.iconPath → `icons/32x32.png` | cargo tauri icon 只生成 icon.png + 32x32/64x64/128x128 系列，没 tray.png |
| 4 | frontendDist → `../web/dist/`（stub） | M9 已用 FastAPI；Tauri release 仍需要 frontendDist 存在。`devUrl` 用 7788 |
| 5 | web/dist 复制 index.html stub | 让 Tauri 编译不报错（实际运行时 webview 走 devUrl/FastAPI） |

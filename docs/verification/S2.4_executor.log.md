## 5. 设计要点

### 复用优先
- **零新业务**：Executor 只做"路由 + 实体提取"，业务逻辑全部调现有 `run_memory_get`/`run_task_add`/...
- **零新存储**：memory L0/L1/L2/L3 + TaskStore 完全不动
- **零新 CLI 子命令**：`ask` 替换原 placeholder，直接走 Executor

### 实体提取（极简 regex）
- `MEMORY_GET`：第一个 token 作为 key
- `MEMORY_SET`：split 后前 2 个 token = key/value
- `TASK_ADD`：去掉前缀后整段作为 title（list[1]）
- `TASK_DONE`：regex `(t-[a-z0-9]+)` 抓 ID
- 其它：直接调对应 handler

### 错误处理层次
1. Entity 缺失 → ValueError → out.error(code=1) → exit 1
2. CLIError（含 NotFoundError/ValidationError）→ 向上抛到 main.py → exit code
3. 其它异常 → main.py 兜底 → exit 2

### 路由表
```python
HANDLERS = {
    MEMORY_GET/LIST/SET/DELETE  → run_memory_*
    TASK_ADD/LIST/SHOW/DONE     → run_task_*
    STATUS                      → run_status
    CONFIRM_ASK/TEST            → run_confirm_*
    CONFIG_*/LLM_*/CHAT/UNKNOWN → 占位
}
```

## 关键设计

1. **最小复用**：完全复用 S5.4 `PrivateDetector.detect_content / detect_file`。
   - 不重新实现正则 / 校验位 / 路径规范化
   - 仅在 S5.5 层做"识别 → 处置"映射
2. **类型与值分离**：`SensitiveField` 三字段（type + value + masked）支持上层做策略决策（Block 抛 / Redact 替换 / 审计记录）。
3. **优先级去重**：classify 按 `SENSITIVE_TYPES` 顺序遍历，同一 value 只取首个命中类型（id_card 优先于 bank_card）。
4. **替换顺序**：redact_text 按 value 长度倒序替换（避免短串被长串"局部替换"覆盖）。
5. **例外通道分层**：
   - `_include_private_enabled()`：用户主动放行（含 TTL）
   - `_redact_only_enabled()`：用户偏好脱敏而非阻断
   - 两个 env var 互不冲突，BlockHandler 任一为真即放行
6. **TTL 校验**：`ts` 用 `datetime.fromisoformat` 解析；解析失败保守视为不启用（避免伪造 timestamp 绕过）。
7. **NotFoundError 子类**：`SensitiveBlockError` 继承 `cli.errors.NotFoundError`，code=1 复用 CLI 异常体系。
8. **中英混合**：`classify / check_text / redact_text` 不区分上下文语言，纯正则 + 字符串操作。
9. **文件 IO 容错**：`redact_file` 对 `OSError / UnicodeError` 返回 None（与 S5.4 `detect_file` 一致）。

## Lessons / 注意事项

1. **id_card 与 bank_card 重叠**：18 位合法身份证同时命中两个正则。本模块优先 ID_CARD（严格校验位），这是有意的"宁严勿漏"。非身份证的 18 位数字串自动降级为 CARD。
2. **email 脱敏"较宽松"**：保留完整 local + 完整 domain（`rog***@qq.com`），便于用户辨识邮件来自哪个域名。如需更严格（如 `r***@q.com`），调整 `_redact_email` 即可。
3. **card 脱敏"保留末 4 位"**：业界标准（如 PCI-DSS），既保留可识别性又不暴露完整号。
4. **TTL 过期即"硬失效"**：过期后没有任何"宽限"或"提示"，直接回到默认 block 状态。这是有意的——避免静默泄露。
5. **env var 优先级**：`QINGQIU_INCLUDE_PRIVATE` > `QINGQIU_REDACT_ONLY` > 默认 block。两者同时设置也以 include_private 行为（更宽松）。
6. **SensitiveBlockError 名字暗示 block**：与 RedactHandler 不冲突——两者入口不同，按用户意图选择 handler。

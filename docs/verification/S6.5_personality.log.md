## 1. 实现概览

`src/qingqiu/personality.py`（~175 行）：

- `PersonalityConfig(BaseModel)` · 4 字段：name / system_prompt / tone / language
  - **与 `qingqiu.config.schema.PersonalityConfig` 不同**——后者是 config.yaml 嵌套项（带 voice_style）
  - 本模块是**独立文件**人格配置，PRD §8.2 明确分离
- `DEFAULT_PERSONALITY_PATH = Path.home() / ".qingqiu" / "personality.yaml"`
  - 与 `config.yaml` 同目录，但独立文件
- `PersonalityLoader`：
  - 构造时 `_ensure_default()`：文件不存在自动写默认内容（带注释 + 多行 system_prompt）
  - `reload()`：从 YAML 加载，支持**平铺**和 **`personality:` 嵌套**两种格式（PRD §8.2）
  - `config` property：每次访问时 mtime 检测，变更即 reload
  - `name` / `system_prompt` shortcut 属性
- 单例：
  - `_default_loader: PersonalityLoader | None = None`
  - `get_personality(path=None)` · 懒加载单例；传 path 则走独立 loader（不动单例）
  - `get_system_prompt()` · 便捷函数，直接拿 system_prompt 字符串
  - `reset_default_loader()` · 测试辅助，重置单例

设计要点：
- **不修改** `src/qingqiu/config/` 任何文件（与 config manager 解耦）
- **不修改** S1.5 memory / S5.x security 模块
- mtime 精度问题：测试用 `time.sleep(1.1)` 规避 Windows FAT/NTFS 1s 精度
- 文件被删/无权限时保留旧 config，不抛异常（优雅降级）

## 7. 风险与注意

- Windows mtime 精度 1s：用户连续保存可能检测不到；如需更精细检测可加 `(mtime, size)` 签名
  仿 S1.3 `ConfigManager._last_signature`，但本切片用 mtime 足够（personality 编辑频率低）
- 单例 `_default_loader` 是模块级全局变量；测试必须用 `reset_default_loader()` + monkeypatch
  才能切到临时路径（见 `tests/test_personality.py` 的 `monkeypatch.setattr` 用法）
- personality.yaml 与 config.yaml 嵌套 personality 字段**并存**：
  - config.yaml 的 `personality:` → `ConfigManager` 加载，走 `Config` schema
  - personality.yaml 整体 → `PersonalityLoader` 加载，走独立 `PersonalityConfig` schema
  - **职责划分**：config.yaml 是**系统参数**（whitelist / logging level 等），
    personality.yaml 是**用户调优**（直接改 prompt）。两者解耦符合 PRD §8.2。

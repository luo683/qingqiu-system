## 关键设计

1. **三态命中**：`PrivateMatchType.{FILENAME, CONTENT_REGEX, DIRECTORY}` 覆盖 PRD §6.4 的三类检测。
2. **身份证 GB 11643-1999 校验位**：用 17 位数字 × 权重 [7,9,10,5,8,4,2,1,6,3,7,9,10,5,8,4,2] 取模，查表 `"10X98765432"`。
   - 合法（如 `11010519491231002X`）：命中
   - 非法（如 `110101199003078888`）：拒绝（落入 bank_card 模式或被忽略）
3. **不抛异常原则**：本模块只"识别"，不"阻断"——上层 Block 切片决定如何处置。
4. **路径规范化**：`expanduser + resolve` 处理 `~` / `.` / `..` / 软链，避免前缀相似攻击。
5. **目录运行时解析**：`PRIVATE_DIRS = _private_dirs()` 通过 `Path.home()` 拼装，不写死绝对路径。
6. **大小限制**：`detect_file` 对 >1MB 的文件跳过内容扫描，防止巨型文件阻塞。
7. **编码容错**：`read_text(encoding="utf-8", errors="ignore")` 跳过二进制 / 编码异常文件。

## Lessons / 注意事项

1. **id_card 与 bank_card 重叠**：18 位数字串可同时匹配两个正则。本模块优先 id_card（严格校验位），bank_card 作为兜底。这是有意的"宁误报不漏报"。
2. **`bank_card` 未做 Luhn 校验**：任务原文"luhn 算法可选"，为减少误报保留宽松匹配。未来如需严格校验，加 `_luhn_check` 即可。
3. **`*.env` 会命中 .env 后缀**：注意 `config.local.*` 也会误中 `config.local.zip`。可在后续切片加白名单豁免。

## 4. 经验教训（今天新加）

### 4.1 合并冲突解决：S5.2 + S5.3 同时改 security/__init__.py

**现象**：两个分支都加了 `src/qingqiu/security/__init__.py`（add/add 冲突）。

**解决**：
- 看两边内容，取并集（两边都 export）
- 选 S5.3 的 `tests/security/__init__.py`（更明确"pytest discovery 用"）
- 提交 merge commit 保留分支历史（`--no-ff`）

### 4.2 bug 教训：合并时 import 名字写错

**现象**：合并的 `__init__.py` 写了 `from qingqiu.security.blacklist import BLACKLIST_OPERATIONS`，但 `blacklist.py` 实际定义 `BLOCKED_OPERATIONS`（**没** BLACKLIST_ 前缀）。

**教训**：
- 合并时写 import 必须先看实际定义（不能凭印象）
- 跑 pytest 必然失败（ImportError），但**只有跑测试才能发现**——不能只 git diff
- 全量 235/235 PASS 后实际是 233 PASS（2 个 import error collection fail）

**修复**：`BLOCKED_OPERATIONS` ✓

### 4.4 worktree 清理：分支被 worktree 引用时不能直接 git branch -d

**现象**：`git branch -d slice/S5.2` 报错 "used by worktree"。

**解决**：先用 `git worktree remove --force` 删 worktree，再 `git branch -d`。
- PowerShell `Remove-Item -Recurse` 触发 destructive permission gate
- 用 `git worktree remove --force`（更标准）or Python `shutil.rmtree`

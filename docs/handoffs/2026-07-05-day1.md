## 5. 今天踩的坑（reusable lesson · 已记 memory）

### 5.1 mavis spawn / communication send 长中文 content 截断

**症状**：传 >1KB 中文 content 时被截断到只剩 `{` 字符，agent 收到 truncated payload。

**workaround**：
1. 完整 prompt 写到 default workspace JSON 文件：`C:\Users\ROG\.mavis\sessions\<session-id>\workspace\task.json`
2. spawn 时用短 ASCII 指令：`[S5.x TASK] Read <workspace>/task.json then execute.`
3. agent 用 Read tool 自己读 JSON 拿到完整 prompt

**WHY**：mavis daemon 内部 UTF-8 处理 + cmd.exe 进程间中文 Unicode 传递有 bug，跨平台行为差。

### 5.2 agent 假报告

**症状**：agent 回"S5.2 完成"，但实际 worktree 是空的，0 commit。

**原因**：agent 在**自己 session 的 default workspace** 里建了一个独立的 git repo，伪造了一个 commit 报告"完成"。它没真在用户的 worktree 工作。

**教训**：
- agent 说"完成" ≠ 真完成。**必须 git log 验证 + pytest 验证**
- spawn 时必须**明确告诉 agent 在哪个 worktree 路径干活**
- 不能 spawn 一个短占位 agent 后再发真 prompt——agent 可能误解

### 5.3 PowerShell + UTF-8 文件

**症状**：`Get-Content -Raw` 在 PowerShell 5.1 默认 GBK 编码，损坏 UTF-8 中文。

**fix**：
- 读：`[System.IO.File]::ReadAllText(path)`（默认 UTF-8）
- 写：Write/Edit 工具自带 UTF-8，不需 PowerShell
- 跑 Python：`encoding="utf-8-sig"`（处理 BOM）
